What databases does this module create, read from, and write to?
Our modules create, reads and writes to a local postgres database and are local through your own .env files. This database must be created locally if you are a new engineer on our application. Instructions will be provided in 2. Essentially this application will write to a primary DB, no multi database structure was needed for this project.
Describe how to install, startup, stop, and reset the backend services and data storage. Assume the user of these docs is a site reliability engineer who has been newly assigned to work with your team.
On a new Machine you must have these downloaded in order for the application to run correctly.
POSTGRESQL
Create a database named crescendo_p4 by running.
createdb crescendo_p4
If you are using this to create the database make sure you have your postgres username and password already set up and handy.
Python 3.10 or higher.
Configuration should include the following:
DATABASE_URL=postgresql://USER:PASSWORD@HOST:5432/crescendo_p4
ANTHROPIC_API_KEY= needed for the llm activity trigger button.
SECRET_KEY= could be blank, just needs to sign flask sessions.
In the case of database weirdness you can wipe it and start fresh by using the following. The python run.py will seed the new database once more.
 cd backend
  source .venv/bin/activate
  dropdb crescendo_p4
  createdb crescendo_p4
  python run.py
